# HA Mock — лёгкий имитатор Home Assistant

Принимает webhook на `/api/webhook/<id>` (как настоящий HA), показывает анимированный шлагбаум в браузере. Для демо/предзащиты — никаких Docker'ов, разворачивается за 30 секунд.

Проектирование и реализация BLE-шлюза для системы контроля доступа транспорта в инфраструктуре умного дома

## Запуск

```powershell
cd C:\Users\andre\ha-mock
pip install -r requirements.txt
python server.py
```

В логе будет:

```
============================================================
 HA Mock — Дашборд шлагбаума
============================================================
  Дашборд:       http://localhost:8123
  С телефона:    http://192.168.1.42:8123
  Webhook URL:   http://192.168.1.42:8123/api/webhook/gate_open
============================================================
```

Откройте дашборд в браузере. Должен загрузиться тёмный интерфейс с шлагбаумом и пустым журналом.

## Проверка webhook

В отдельном PowerShell:

```powershell
curl.exe -X POST http://localhost:8123/api/webhook/gate_open `
  -H "Content-Type: application/json" `
  -d '{\"vehicle\":\"Андрей\",\"major\":1,\"rssi\":-52}'
```

В браузере должно произойти:
1. Шлагбаум поднимается (анимация 1.5 сек).
2. Машина проезжает (анимация SVG).
3. Через 5 секунд шлагбаум опускается.
4. В журнале — новая зелёная строка «Открыто: Андрей · -52 dBm».
5. Счётчик «Открытий сегодня» → +1.

## С телефона

В Flutter BLE Beacon → вкладка «Шлюз»:
- HA URL: `http://192.168.1.42:8123` (IP вашего ноута)
- Webhook ID: `gate_open`

## Endpoints

| Метод | URL | Что делает |
|---|---|---|
| `GET` | `/` | Дашборд |
| `POST` | `/api/webhook/gate_open` | Принимает запрос от BLE-Gateway, открывает шлагбаум |
| `POST` | `/api/test/open` | Ручное открытие из дашборда |
| `POST` | `/api/test/reset` | Сброс счётчиков и журнала |
| `WS`   | `/ws` | WebSocket для live-обновлений UI |
