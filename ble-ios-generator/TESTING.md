## 1. Собрать и запустить на iPhone (кратко)

Сжатый прогон [Шагов 1–5 README](README.md#сборка-в-xcode-пошагово) — детали и
скриншоты там же:

1. Xcode → **New Project ▸ iOS ▸ App**, имя `BLEGenerator`, Interface **SwiftUI**.
2. Заменить `BLEGeneratorApp.swift` и `ContentView.swift` файлами из `BLEGenerator/`,
   добавить `BeaconAdvertiser.swift` (Copy items if needed, таргет отмечен).
3. Target ▸ **Info** → добавить **Privacy – Bluetooth Always Usage Description**
   (`NSBluetoothAlwaysUsageDescription`). **Без этого ключа приложение падает.**
4. **Signing & Capabilities** → Automatically manage signing → выбрать свой **Team**.
5. Выбрать подключённый iPhone, **Run (⌘R)**. На iPhone один раз: включить
   **Режим разработчика** (Настройки ▸ Конфиденциальность), затем **доверять**
   профилю (Настройки ▸ Основные ▸ VPN и управление устройством).

## 3. Тест-кейсы

Прогоняй по порядку, отмечая результат. Обязательные для вывода о
работоспособности — **ТК-1, ТК-2, ТК-3, ТК-6**; остальные подтверждают
задокументированные ограничения Apple.

### ТК-1 — Запуск без краша
**Шаги:** открыть приложение на iPhone, разрешить доступ к Bluetooth.
**Ожидаемо:** приложение **не падает**; индикатор **синий**, статус
*«Готов к вещанию»*.
> Краш сразу при запуске ⇒ не добавлен `NSBluetoothAlwaysUsageDescription` (шаг 1.3).

### ТК-2 — Старт вещания
**Шаги:** нажать **«Начать вещание»**.
**Ожидаемо:** индикатор **зелёный**, статус *«Вещает»*, ниже показан **UUID**
(моноширинным). Кнопка меняется на *«Остановить»*, поля UUID/имя заблокированы.

### ТК-3 — Приём сторонним сканером *(ключевой)*
**Шаги:** смотреть в таблицу `ble_scanner.py` на Windows.
**Ожидаемо:** за ~1–5 с появляется строка — **Тип = `Generic`**, **Имя = `HA-Tag`**
(или твоё), **Параметры = `Services=<тот самый UUID>`**. При сближении iPhone со
сканером **RSSI растёт** (становится менее отрицательным).
> Это и есть доказательство, что метка реально излучается и опознаётся **по UUID**.
> Строк iBeacon/Eddystone не будет — это норма (iOS их вещать не умеет).

### ТК-4 — Идентификация по UUID, а не по MAC
**Шаги:** оставить вещание на 1–2 мин, наблюдать столбец **Адрес**.
**Ожидаемо:** Service UUID стабилен, а **MAC-адрес может смениться** (iOS
рандомизирует). Вывод: ловить метку нужно по UUID.
> Может не проявиться за короткий тест — необязательный.

### ТК-5 — Фон / overflow *(проверка ограничения)*
**Шаги:** свернуть приложение (кнопка Home / свайп), подождать ~15–20 с, смотреть сканер.
**Ожидаемо:** строка метки **исчезает** из таблицы Windows-сканера (UUID ушёл в
overflow-область, виден только Apple-устройствам). Вернуть приложение на передний
план → метка **снова появляется**.
> Подтверждает: для стороннего приёмника держать приложение на переднем плане.

### ТК-6 — Остановка
**Шаги:** нажать **«Остановить»**.
**Ожидаемо:** статус → *«Готов к вещанию»* (синий); строка метки **пропадает** из
сканера в течение **≤15 с** (stale-таймаут сканера). Поля снова редактируемы.

### ТК-7 — Смена UUID *(опционально)*
**Шаги:** кнопкой **↻** сгенерировать новый UUID, снова «Начать вещание».
**Ожидаемо:** в сканере поле `Services=` показывает **новый** UUID.

### ТК-8 — Запуск на симуляторе *(негативный, опционально)*
**Шаги:** запустить схему на **iOS Simulator**.
**Ожидаемо:** индикатор **красный**, статус *«BLE-вещание недоступно (симулятор…)»*.
> Подтверждает корректную обработку отсутствия BLE и объясняет, почему нужен реальный iPhone.

## 4. Если что-то не так

| Симптом | Причина | Что делать |
|---------|---------|------------|
| Краш при запуске | Нет `NSBluetoothAlwaysUsageDescription` | Добавить ключ (шаг 1.3) |
| Статус «BLE-вещание недоступно» | Запущено на симуляторе | Запустить на реальном iPhone |
| Статус «Нет доступа к Bluetooth» | Не выдано разрешение | Настройки ▸ приложение ▸ Bluetooth → вкл |
| Метки нет в сканере | Приложение свёрнуто / далеко / BT Windows выкл / UUID не тот | Передний план; поднести; включить BT; сверить строку UUID глазами |
| Метка видна только пока приложение открыто | Норма (overflow в фоне) | Держать на переднем плане; для постоянной метки — ESP32/ESPHome |
| Сканер: «Bluetooth выключен» | Выключен BT на Windows | Включить Bluetooth в настройках Windows |
