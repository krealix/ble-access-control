# Транспорты подсистемы «Шлюз»: HTTP, TCP, MQTT

Как настроить доставку команды открытия из вкладки «Шлюз» во внешнюю систему
(умный дом, микроконтроллер, брокер) и как проверить каждый транспорт.

---

## Общее

При срабатывании (метка опознана и достаточно близко) шлюз формирует единый
объект данных и доставляет его выбранным транспортом:

```json
{
  "vehicle": "Машина Иванова",
  "uuid": "E2C56DB5DFFB48D2B060D0F5A71096E0",
  "mac": "C0:1A:...",
  "major": 1,
  "minor": 7,
  "rssi": -52,
  "timestamp": "2026-06-12T14:03:11.482"
}
```

В текстовых шаблонах (TCP, формат `text`) доступны плейсхолдеры:
`{vehicle}` `{uuid}` `{mac}` `{major}` `{minor}` `{rssi}` `{timestamp}`.

Результат доставки фиксируется в журнале шлюза: зелёная строка — успех,
красная — ошибка.

---

## HTTP — Home Assistant и веб‑хуки

**Когда использовать:** есть умный дом с HTTP‑эндпоинтом (Home Assistant,
Node‑RED) или собственный веб‑сервер. Самый быстрый старт.

**Поля в UI:**

| Поле | Пример | Назначение |
|------|--------|-----------|
| Адрес HA | `http://192.168.0.10:8123` | базовый адрес сервера |
| Webhook ID | `gate_open` | идентификатор веб‑хука |

Итоговый запрос: **POST** на `{Адрес HA}/api/webhook/{Webhook ID}`,
заголовок `Content-Type: application/json`, тело — объект из раздела «Общее».

**Пример — автоматизация Home Assistant:**
```yaml
automation:
  - alias: Открытие шлагбаума по BLE
    trigger:
      - platform: webhook
        webhook_id: gate_open
        allowed_methods: [POST]
        local_only: true
    action:
      - service: cover.open_cover
        target: { entity_id: cover.shlagbaum }
```
Данные метки доступны как `{{ trigger.json.vehicle }}`, `{{ trigger.json.rssi }}`.

**Пример — проверка без HA (приёмник на ПК).** Адрес в шлюзе `http://<IP-ПК>:8000`:
```python
from http.server import BaseHTTPRequestHandler, HTTPServer
class H(BaseHTTPRequestHandler):
    def do_POST(self):
        n = int(self.headers.get('Content-Length', 0))
        print('ОТКРЫТИЕ →', self.rfile.read(n).decode())
        self.send_response(200); self.end_headers()
HTTPServer(('0.0.0.0', 8000), H).serve_forever()
```

**Журнал:** `HTTP: 200 OK` · ошибки: `HTTP: 404 Not Found`, `HTTP: таймаут (5 сек)`.

## TCP — микроконтроллеры и «сырые» команды

**Когда использовать:** открываете реле‑модуль / ESP32 / Arduino / PLC,
слушающий TCP‑сокет; или нужно отправить точную бинарную команду контроллеру.

**Поля в UI:** `Хост`, `Порт` (напр. `9999`), `Формат`, `Шаблон`.

**Три формата payload:**

| Формат | Что отправляется | Пример → результат |
|--------|------------------|--------------------|
| `json` | весь объект + `\n` | `{"vehicle":"…","rssi":-52,…}` |
| `text` | шаблон с подстановкой плейсхолдеров | `OPEN {vehicle}\n` → `OPEN Машина Иванова⏎` |
| `hex`  | фиксированные байты из hex‑строки | `87 00 00 00 00 00 00 00 77 02` → 10 байт |

> Формат `hex` удобен, чтобы отправить ровно тот 10‑байтный STOWN‑пакет,
> который понимает контроллер шлагбаума. Подстановка плейсхолдеров в `hex`
> не выполняется — команда статична.

**Пример — приём на ПК:**
```
ncat -lk 9999          # печатает каждую пришедшую команду
```

**Пример — приёмник на Python:**
```python
import socket
srv = socket.socket(); srv.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
srv.bind(('0.0.0.0', 9999)); srv.listen()
print('Жду команды на :9999')
while True:
    conn, addr = srv.accept()
    print('ОТКРЫТИЕ от', addr, '→', conn.recv(1024)); conn.close()
```

**Журнал:** `TCP: 18 байт → 192.168.0.10:9999` · ошибки: `TCP: таймаут…`, `TCP: <причина>`.

## MQTT — брокерная интеграция

**Когда использовать:** в доме есть MQTT‑брокер (Mosquitto), к которому
подключены Home Assistant / Node‑RED / ESPHome; одно событие нужно нескольким
подписчикам.

**Поля в UI:**

| Поле | Пример | Примечание |
|------|--------|-----------|
| Брокер | `192.168.0.10` | хост брокера |
| Порт | `1883` | стандартный MQTT |
| Топик | `home/gate/open` | куда публикуем |
| Логин / Пароль | *(пусто)* | если брокер без авторизации — оставить пустым |

Шлюз публикует JSON‑объект (см. «Общее») в указанный топик, QoS 0.

**Пример — наблюдать сообщения:**
```
mosquitto_sub -h 192.168.0.10 -t home/gate/open -v
```

**Пример — автоматизация Home Assistant:**
```yaml
automation:
  - alias: Открытие по MQTT
    trigger:
      - platform: mqtt
        topic: home/gate/open
    action:
      - service: cover.open_cover
        target: { entity_id: cover.shlagbaum }
```

**Журнал:** `MQTT: → 192.168.0.10:1883 / home/gate/open` ·
ошибки: `MQTT: таймаут подключения`, `MQTT: не подключиться…`.

## Как выбрать транспорт

| Транспорт | Берите, если… |
|-----------|----------------|
| **HTTP** | есть Home Assistant или простой веб‑сервер — самый быстрый старт |
| **MQTT** | дом уже на брокере, на одно событие нужно несколько подписчиков |
| **TCP**  | открываете напрямую микроконтроллер/реле или шлёте «сырой» байт‑код |

Во всех случаях факт доставки виден зелёной строкой в журнале шлюза — это и есть
подтверждение, что команда ушла во внешнюю систему. Для отладки сети помогает
встроенная вкладка «Сканер» (метка в эфире) и локальные приёмники из примеров выше.
